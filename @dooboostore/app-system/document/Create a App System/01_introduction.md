# 서문: 왜 공통 애플리케이션 시스템을 만드는가?

현대 웹 애플리케이션 개발은 복잡하고 다양한 요구사항을 가집니다. 사용자 인터페이스, 데이터 통신, 상태 관리, 사용자 피드백 등 여러 영역에 걸쳐 공통적으로 필요한 기능들이 존재합니다. 이러한 기능들을 매번 새로 구현하는 것은 비효율적이며, 코드의 일관성을 해치고 유지보수 비용을 증가시킵니다.

이 책에서는 `@dooboostore/app-system`이라는 공통 애플리케이션 시스템의 설계와 구현 과정을 다룹니다. 이 시스템은 `@dooboostore` 생태계의 다른 프레임워크들(simple-boot, dom-render 등)과 연동하여, 웹 애플리케이션 개발에 필요한 공통적이고 횡단적인 기능들을 모듈화하여 제공합니다.

## `@dooboostore/app-system`의 핵심 특징

-   **ApiService:** 계층적 구조를 가진 강력한 HTTP 클라이언트로, API 호출, 에러 관리, 진행 상황 추적, 알림 기능 등을 통합합니다.
-   **Alert System:** 사용자에게 알림, 에러 메시지 등을 표시하는 유연한 시스템을 제공합니다.
-   **UI Components:** 재사용 가능한 기본 UI 컴포넌트들을 제공하여 개발 생산성을 높입니다.
-   **Proxy System:** API 요청을 가로채어 수정하거나 확장하는 기능을 제공합니다.
-   **Store:** 애플리케이션의 상태를 예측 가능하고 효율적으로 관리하는 기능을 제공합니다.

## 이 책에서 다룰 주요 내용

-   **시스템의 기반:** `ApiService`를 통한 HTTP 통신과 인터셉터 패턴.
-   **사용자 피드백:** `Alert` 시스템의 역할과 구현.
-   **재사용 가능한 UI 컴포넌트:** 내장 컴포넌트의 설계와 활용.
-   **상태 관리와 프록시 시스템:** 애플리케이션 상태 관리와 API 요청 가로채기.
-   **프레임워크 연동과 확장성:** `Simple-Boot` 생태계와의 연동 및 시스템 확장성 설계.

이 여정을 통해 여러분은 복잡한 애플리케이션의 공통 기능들을 어떻게 체계적으로 설계하고 구현하는지 이해하게 될 것입니다. 이제, `@dooboostore/app-system`의 설계와 구현 여정을 시작해 봅시다.
