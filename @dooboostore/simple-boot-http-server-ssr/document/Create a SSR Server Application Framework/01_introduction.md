# 서문: 왜 서버사이드 렌더링 프레임워크를 만드는가?

현대 웹 애플리케이션은 사용자 경험과 검색 엔진 최적화(SEO)라는 두 가지 중요한 요구사항을 동시에 충족해야 합니다. 단일 페이지 애플리케이션(SPA)은 풍부한 사용자 경험을 제공하지만, 초기 로딩 속도와 SEO 측면에서 불리할 수 있습니다. 서버사이드 렌더링(SSR)은 이러한 SPA의 단점을 보완하며, 초기 로딩 성능을 향상시키고 검색 엔진 크롤러가 콘텐츠를 더 잘 이해할 수 있도록 돕습니다.

이 책에서는 `@dooboostore/simple-boot-http-server-ssr`이라는 프레임워크의 설계와 구현 과정을 다룹니다. 이 프레임워크는 `@dooboostore/simple-boot-front` (프론트엔드 프레임워크)와 `@dooboostore/simple-boot-http-server` (HTTP 서버 프레임워크)를 통합하여, Node.js 환경에서 SSR 애플리케이션을 효율적으로 구축할 수 있도록 돕습니다.

## `Simple-Boot-HTTP-SSR`의 핵심 특징

-   **서버사이드 렌더링:** 클라이언트 측 애플리케이션을 서버에서 미리 렌더링하여 초기 HTML을 생성합니다.
-   **JSDOM 통합:** 서버 환경에서 브라우저와 유사한 DOM 환경을 제공하여 프론트엔드 코드를 실행할 수 있게 합니다.
-   **컴포넌트 풀링:** `SimpleBootFront` 인스턴스를 풀(Pool)로 관리하여 동시 SSR 요청을 효율적으로 처리합니다.
-   **데이터 하이드레이션:** 서버에서 가져온 데이터를 클라이언트로 전달하고, 클라이언트에서 중복 API 호출 없이 UI를 활성화합니다.
-   **통합된 코드베이스:** 서버와 클라이언트 간에 라우팅, 서비스, 컴포넌트 코드를 공유하여 개발 효율성을 높입니다.
-   **Intent 기반 통신:** 클라이언트-서버 간의 유연한 데이터 통신 메커니즘을 제공합니다.

## 이 책에서 다룰 주요 내용

-   **SSR의 기본:** `SimpleBootHttpSSRServer`의 역할과 JSDOM을 활용한 서버 측 렌더링 환경 구축.
-   **SSR 필터:** 요청 처리 파이프라인에서 SSR을 수행하는 핵심 필터의 동작 원리.
-   **데이터 하이드레이션:** 서버에서 렌더링된 데이터를 클라이언트에서 재활용하는 메커니즘.
-   **클라이언트-서버 통신:** Intent 기반으로 SSR 환경에서 클라이언트와 서버가 통신하는 방법.
-   **프레임워크의 기반:** `Simple-Boot Core` 및 `Simple-Boot-Front`와의 연동과 SSR 환경에 최적화된 통합.

이 여정을 통해 여러분은 SSR 애플리케이션의 복잡한 세계를 탐험하고, 고성능의 SEO 친화적인 웹 애플리케이션을 설계하는 데 필요한 지식을 얻게 될 것입니다. 이제, `Simple-Boot-HTTP-SSR`의 설계와 구현 여정을 시작해 봅시다.
